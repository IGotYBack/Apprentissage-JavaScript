FAQ - Ce qu'on sait de Git et Github

Ceci est un résumé très grossier des premières notions apprises sur Git et Github.
Je suis actuellement en apprentissage total et des erreurs peuvent se glisser dans mes écrits, voire ma compréhension.

Git, à quoi ça sert ?

Git est un système de contrôle des versions.

C'est-à-dire :

  • c'est un système qui va enregistrer les modifications apportées à un document et les compartimenter en diverses versions, en fonction des breakpoint enregistrés
  • on va pouvoir faire appel à n'importe laquelle des versions enregistrée, à n'importe quel moment
  • plusieurs personnes vont pouvoir facilement collaborer sur un projet et avoir leur propre version du projet sur leur ordinateur

Github, à quoi ça sert ?

Service qui nous permet de stocker des répertoires en ligne et qui va grandement faciliter le travail de groupe sur un même projet.

En effet, Github permet de stocker le projet sur lequel on travaille et, chaque personne participant au projet, va pouvoir le pull de Github pour le modifier localement.

Ensuite, il est possible de push la modification dans le répertoire du projet et de merge la modification avec le projet après vérification de la modification par les autres participants.

Comment Git fonctionne ?

Git va nous permettre de travailler sur une version de notre projet, dans la main branch ou dans une branch à part afin de ne pas toucher à notre code source lorsqu'on effectue des modifications.

Pour pouvoir effectuer une modification à la version sur laquelle on travaille il va falloir effectuer plusieurs opérations :

  • Modifier le document
  • Passer les modifications à l'état de "staging" (révision ?)
  • Fusionne les modifications avec la version précédente
  • Fusionner la branch au main, si on a travaillé sur une branch secondaire du projet

Ca nous permet de fragmenter les modifications et de garder une trace concise de celles-ci.

Quelles sont les commandes principales pour Git ?

Git init qui nous permet d'initialiser un projet Git.

Git add *nom du fichier* pour ajouter un fichier à l'état "staging" ou Git add *.* pour y ajouter tous les fichiers.

Git commit pour fusionner la modification avec la version précédente.

Git checkout pour aller voir une branch ou un log spécifique

Git checkout peut également être utilisé pour créer de nouvelles branch
Git checkout -b *nom de la nouvelle branch*

Git log ou log --oneline pour vérifier l'historique de modifications

Quelles sont les commandes principales pour Github ?

Git clone pour cloner un répertoire de Github sur notre ordinateur.

Git pull *origin/alias* *branch* pour aller chercher des fichiers sur Github.

Git push *code du répertoire* *branch* pour pousser une version qui n'a pas été pull de Github.

Git remote add *alias* *code du répertoire* pour donner un alias au lien du répértoire.

Avec Visual Studio Code, il est possible de faire les différentes étapes de staging, commit et push directement depuis l'onglet *source control*.
